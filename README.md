# AI reStarter (Rust版)

AI reStarter は、AI 統合 IDE を監視するために設計された高性能な自動復旧システムです。画面キャプチャとテンプレートマッチングを使用してエラーや停止状態を検出し、復旧コマンドを自動的に実行して継続的な動作を保証します。

最大のパフォーマンス、最小のリソース使用量、および信頼性を実現するために **Rust** で書き直されました。

## 機能

- **高性能**: Rust で構築されており、起動は 100ms 未満、メモリフットプリントも低く抑えられています。
- **画面監視**: 画面の状態を効率的にキャプチャして問題を検出します。
- **テンプレートマッチング**: 画像テンプレートを使用してエラー状態を正確に特定します。
- **自動復旧**: クリックやキーボード操作を自動的に実行してワークフローを復元します。
- **システムトレイ**: クイックアクセスコントロールを備えた、邪魔にならないバックグラウンド動作。

## 前提条件

- **Rust ツールチェーン**: 1.82 以降。[Rust をインストール](https://www.rust-lang.org/tools/install)

## インストールとビルド

1. **リポジトリのクローン**
   ```bash
   git clone <repository-url>
   cd AI_reStarter
   ```

2. **プロジェクトのビルド**
   ```bash
   cargo build --release
   ```
   バイナリは `target/release/ai-restarter.exe` に配置されます。

## 使い方

### ソースから実行
```bash
cargo run
```

### 設定
アプリケーションは設定に `config.toml` を使用します。
- **Monitor Interval (監視間隔)**: 画面をチェックする頻度を調整します。
- **Templates (テンプレート)**: 検索する画像と、見つかったときに実行するアクションを定義します。

### 動作
- **監視**: アプリはバックグラウンドで動作します。ステータスはシステムトレイアイコンで確認してください。
- **自動復旧**: マッチするテンプレート（例：エラーダイアログ）が見つかると、設定されたアクション（クリック/キー押下）が自動的に実行されます。

## プロジェクト構成
- `src/core`: モニタリングとアクションのコアロジック。
- `src/ui`: ユーザーインターフェイスとシステムトレイの実装。
- `src/config`: 設定管理。
